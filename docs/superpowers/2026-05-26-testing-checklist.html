③ Email Tracking — opens & clicks Needs ngrok

Open + click tracking only works when the recipient's mail client can reach your API over the public internet. On localhost the tracking pixel/redirect is unreachable, so events never record — that's why your earlier test showed nothing.

Prerequisite — expose your API publicly:
  1. In one terminal: ngrok http 3001 (the API port — NOT 5173, that's the web app). Copy the https://….ngrok-free.dev URL.
  2. In apps/api/.env set PUBLIC_API_URL=https://<your-ngrok>.ngrok-free.dev (no trailing slash) and make sure EMAIL_TRACKING_SECRET is set (64 hex chars).
  3. Restart pnpm dev so the API picks up the env.
  4. Open the web app at http://localhost:5173 (your own browser — not the ngrok URL).

CC/BCC: in the compose dialog click "+ Cc · Bcc", add an address you control, send, and confirm it arrives at the CC address.
