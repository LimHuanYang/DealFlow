🔁 How it works (safe by design)
type query→ POST /ai/nl-search {query, entity}→ AI (per-org provider)→ returns constrained FilterDSL (JSON)→ Zod allowlist validate→ existing repo query · org-scoped · role-gated→ results + chips

The AI never touches the database. It only emits a small JSON FilterDSL from an allow-list of fields/operators; the server Zod-validates it and runs the SAME list query the manual filters use. No raw SQL, no data leakage, tenancy enforced server-side. Invalid/over-broad → reject + ask to rephrase (or keyword fallback).
